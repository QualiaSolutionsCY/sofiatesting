2. Attributes Configuration
These fields represent direct data about the land.
A. Mandatory Attributes
Constraint: These fields must be present in the payload.
title (string)
The name of the listing. Max 255 characters.

body (object)
The main description. Must contain the property value.
Example: {"value": "A beautiful plot of land..."}

field_price (number)
The cost of the land.

field_land_size (integer)
The size of the land in square meters.

B. Optional Attributes (Land Details)
Constraint: Populate these if data is available to increase listing quality.
field_building_density (integer): Percentage of density allowed.
field_site_coverage (integer): Percentage of site coverage allowed.
field_floors (integer): Maximum number of floors allowed.
field_height (number): Maximum building height allowed.
field_video_walkthrough (string): URL to a video.
field_is_potentially_sold (boolean): Default is false.
field_notes (object): Internal notes. Requires value.
field_own_reference_id (string): External ID.
field_phone_number (string): Contact number.
field_metatag (object): SEO tags.

C. AI Workflow Attributes
Constraint: Sophia AI must populate these to track the generation process.
field_ai_generated (boolean): Set to true.
field_ai_state (string): Set to "draft".
field_ai_message (object): Summary of generation. Requires value.
Example: {"value": "Draft generated based on user prompt."}

field_ai_probably_exists (boolean): Set to true if a similar listing was found, otherwise false.
field_ai_draft_own_reference_id (string): Internal AI reference ID.

D. System Attributes (Read-Only)
Constraint: Do not send these fields. They are generated by the server.

3. Relationships & Taxonomy Endpoints
Instruction: For every relationship below, Sophia AI must perform a GET request to the provided Endpoint Route to fetch the available options. Use the retrieved id (UUID) and type to construct the relationship object.
A. Mandatory Relationships
field_land_type
Description: The classification of the land (e.g., Plot, Field).
Resource Type: taxonomy_term--land_type
Endpoint Route: /jsonapi/taxonomy_term/land_type

field_listing_type
Description: The status of the listing (e.g., For Sale, For Rent).
Resource Type: taxonomy_term--listing_type
Endpoint Route: /jsonapi/taxonomy_term/listing_type
Default UUID: 8f187816-a888-4cda-a937-1cee84b9c0ee

field_land_price_modifier
Description: Price context (e.g., VAT included).
Resource Type: taxonomy_term--price_modifier
Endpoint Route: /jsonapi/taxonomy_term/price_modifier
Default UUID: ab39af2d-c8f5-4971-9fa5-2df6822ab9a9

field_location
Description: The geographical location entity.
Resource Type: Varies (Check endpoint response).
Endpoint Route: /jsonapi/node/location (or relevant location endpoint).

field_land_gallery
Description: Image files for the listing.
Resource Type: file--file
Endpoint Route: Requires file upload to /jsonapi/node/land/field_land_gallery first, or referencing existing file UUIDs.
Note: Must be an array.


B. Optional Relationships
field_land_title_deed
Description: Title deed status.
Resource Type: taxonomy_term--title_deed
Endpoint Route: /jsonapi/taxonomy_term/title_deed
Default UUID: 5c553db1-e53d-46a2-b609-093d17e75a7a

field_infrastructure
Description: Utilities available (e.g., Water, Electricity).
Resource Type: taxonomy_term--infrastructure_
Endpoint Route: /jsonapi/taxonomy_term/infrastructure_
Note: This is an array (multiple values allowed).

field_land_views
Description: Scenery views (e.g., Sea View, Mountain View).
Resource Type: taxonomy_term--property_views
Endpoint Route: /jsonapi/taxonomy_term/property_views
Note: This is an array.

field_marketing_agreement
Resource Type: file--file
Note: Document file upload.

field_other_document
Resource Type: file--file
Note: Document file upload.

field_title_deed_file
Resource Type: file--file
Note: Document file upload.

field_ai_listing_instructor / field_ai_listing_reviewer
Resource Type: user--user
Note: References to system users.
