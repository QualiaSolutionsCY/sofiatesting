  <name>Task 2: Create validation wrapper with Zod integration</name>
  <files>supabase/functions/sophia-bot/tools/validation.ts</files>
  <action>
Create validation module that:

1. Import ZodSchema from zod
2. Import TOOL_SCHEMAS from "./schemas.ts"
3. Define ValidationResult type:
   ```typescript
   export type ValidationResult =
     | { valid: true; data: Record<string, unknown> }
     | { valid: false; error: string; issues?: string[] }
   ```
4. Export validateToolArguments function:
   ```typescript
   export function validateToolArguments(
     toolName: string,
     args: Record<string, unknown>
   ): ValidationResult
   ```
5. Implementation:
   - Look up schema from TOOL_SCHEMAS by toolName
   - If schema not found, return { valid: false, error: `Unknown tool: ${toolName}` }
   - Call schema.safeParse(args)
   - If success, return { valid: true, data: result.data }
   - If failure, return { valid: false, error: "Invalid arguments", issues: result.error.issues.map(i => `${i.path.join('.')}: ${i.message}`) }
6. Add JSDoc comments explaining purpose and usage

Use existing error handling patterns from executor.ts.

Modify executeTool function to validate arguments before dispatch:

1. Import validateToolArguments from "./validation.ts"
2. At start of executeTool (after timer but before switch), add validation:
   ```typescript
   // Validate tool arguments against schema (SEC-04)
   const validation = validateToolArguments(tool.name, tool.arguments);
   if (!validation.valid) {
     logger.warn("Tool validation failed", {
       category: LogCategory.TOOL,
       toolName: tool.name,
       error: validation.error,
       issues: validation.issues,
     });
     return {
       error: `Invalid arguments for ${tool.name}: ${validation.error}`,
       retryable: false
     };
   }

   // Use validated data (type-safe)
   const validArgs = validation.data;
   ```
