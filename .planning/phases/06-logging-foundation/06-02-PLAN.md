serve(async (req) => {

    category: LogCategory.WEBHOOK,
  };

  return withContext(context, async () => {

      category: LogCategory.WEBHOOK,
      method: req.method,
      url: req.url,
    });

    try {
      // ... existing request handling ...

      const duration = Date.now() - startTime;
      logger.info("Webhook request completed", {
        category: LogCategory.WEBHOOK,
        duration,
      });

      return response;
    } catch (error) {
      const duration = Date.now() - startTime;
      logger.error("Webhook request failed", error, {
        category: LogCategory.WEBHOOK,

      });
      throw error;
    }
  });
});

  <action>
Systematically replace all console.log/error/warn calls with structured logger:

**Pattern mapping:**

| Old Pattern | New Pattern | Category |
|-------------|-------------|----------|
| `console.log("[Email Detection]", ...)` | `logger.debug("Email detection: ...", { category: LogCategory.EMAIL, ... })` | EMAIL |
| `console.log("[SOPHIA]", ...)` | `logger.info("...", { category: LogCategory.AI, ... })` | AI |
| `console.log("[IMAGE]", ...)` | `logger.info("...", { category: LogCategory.IMAGE, ... })` | IMAGE |
| `console.log("[Tool]", ...)` | `logger.info("...", { category: LogCategory.TOOL, ... })` | TOOL |
| `console.log("[DOCX]", ...)` | `logger.info("...", { category: LogCategory.DOCX, ... })` | DOCX |
| `console.log("[Webhook]", ...)` | `logger.info("...", { category: LogCategory.WEBHOOK, ... })` | WEBHOOK |
| `console.log("[Memory]", ...)` | `logger.debug("...", { category: LogCategory.MEMORY, ... })` | MEMORY |
| `console.error(...)` | `logger.error("...", error, { category: ..., errorCategory: categorizeError(error) })` | varies |
| `console.warn(...)` | `logger.warn("...", { category: ... })` | varies |

**Key replacements:**

1. Email detection logs (lines 114-117): Use LogCategory.EMAIL, level debug
2. AI response logs: Use LogCategory.AI
3. Image processing logs: Use LogCategory.IMAGE
4. Tool execution logs: Use LogCategory.TOOL
5. DOCX generation logs: Use LogCategory.DOCX
6. Error handling: Always include errorCategory via categorizeError()

**Important:**
- Do NOT remove information from logs, just restructure into context fields
- Debug-level logs for verbose internal state (email detection patterns, etc.)
- Info-level logs for significant events (request start, tool calls, etc.)
- Warn-level logs for unexpected but handled situations
- Error-level logs for failures

**PII handling:**
- Logger already redacts phone numbers and emails via redactPII()
- Do not manually redact - let the logger handle it
