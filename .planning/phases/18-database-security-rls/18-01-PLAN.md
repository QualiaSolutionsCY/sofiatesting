  <action>
Create a new migration file that enables RLS and creates policies for 7 core web app tables:

**Tables to protect:**
1. **User** - Users can only read/update their own row
2. **Chat** - Users can only access chats they own (userId = auth.uid())
3. **Message_v2** - Users can only access messages in their chats (via Chat.userId = auth.uid())
4. **Vote_v2** - Users can only vote on messages in their chats (via Chat.userId = auth.uid())
5. **Suggestion** - Users can only access suggestions they created (userId = auth.uid())
6. **Document** - Users can only access documents they created (userId = auth.uid())
7. **DocumentSend** - Users can only access document sends they created (userId = auth.uid())

**For each table:**
```sql
-- Enable RLS
ALTER TABLE "TableName" ENABLE ROW LEVEL SECURITY;

-- Policy for SELECT (read)
CREATE POLICY "Users can view own records"
  ON "TableName" FOR SELECT
  USING (userId = auth.uid());

-- Policy for INSERT (create)
CREATE POLICY "Users can insert own records"
  ON "TableName" FOR INSERT
  WITH CHECK (userId = auth.uid());

-- Policy for UPDATE (modify)
CREATE POLICY "Users can update own records"
  ON "TableName" FOR UPDATE
  USING (userId = auth.uid())
  WITH CHECK (userId = auth.uid());

-- Policy for DELETE (remove)
CREATE POLICY "Users can delete own records"
  ON "TableName" FOR DELETE
  USING (userId = auth.uid());
```

**Special cases:**
- **User table**: No userId column, use `id = auth.uid()` directly
- **DocumentSend**: Has userId column, same pattern as Document (userId = auth.uid())
- **Message_v2 and Vote_v2**: Join through Chat table to check ownership:
  ```sql
  USING (
    chatId IN (
      SELECT id FROM "Chat" WHERE userId = auth.uid()
    )
  )
  ```

**Migration structure:**
- Start with comment header explaining purpose
- Enable RLS on all 7 tables first
- Create policies for each table grouped by table
- Use consistent naming: "Users can [action] own [records]"

**Important:**
- This protects against web app users accessing other users' data
- Edge Functions use service_role key which bypasses RLS (unchanged)
- Admin panel queries will need to handle RLS (Phase 19 will add admin role checks)
