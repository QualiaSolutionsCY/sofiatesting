  <name>Task 1: Create exponential backoff retry utility</name>
  <files>supabase/functions/sophia-bot/utils/retry.ts</files>
  <action>
Create a new file `retry.ts` with exponential backoff retry logic:

1. Define `RetryConfig` interface:
   ```typescript
   interface RetryConfig {
     maxRetries?: number;      // Default: 3
     baseDelayMs?: number;     // Default: 1000
     maxDelayMs?: number;      // Default: 10000
     jitterMs?: number;        // Default: 500 (random 0-500ms added)
     retryableStatuses?: number[]; // Default: [408, 429, 500, 502, 503, 504]
   }
   ```

2. Create `withRetry<T>` async function that:
   - Takes an async operation function `() => Promise<T>`
   - Takes optional `RetryConfig` and `operationName` string
   - Calculates delay as: `min(maxDelay, baseDelay * 2^attempt) + random(0, jitter)`
   - Logs each retry attempt with logger.warn including:
     - operationName
     - attempt number (1-indexed)
     - delay before next retry
     - error message
     - category: LogCategory.GENERAL
   - Re-throws after max retries exhausted
   - Returns result of successful operation

3. Create `isRetryableError` helper that checks:
   - Network errors (fetch failures, timeouts)
   - HTTP status codes in retryableStatuses
   - Error message patterns ("ECONNREFUSED", "timeout", "network")

4. Export: `withRetry`, `RetryConfig`, `isRetryableError`

Use Deno imports for dependencies. Import logger from `./logger.ts`.
  </action>
  <verify>File exists and exports withRetry, RetryConfig, isRetryableError</verify>
  <done>Retry utility with exponential backoff + jitter ready for API client integration</done>
