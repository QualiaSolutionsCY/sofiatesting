Replace manual confirmDelete check with Zod schema validation.

Steps:
1. Import Zod at top: `import { z } from "zod";`
2. Define confirmation schema before DELETE handler:
```typescript
const confirmationSchema = z.object({
  confirmDelete: z.literal(true, {
    errorMap: () => ({
      message: "To delete your account, you must send { confirmDelete: true }"
    })
  })
});
```
3. Replace lines 44-55 (manual validation) with:
```typescript
const bodyParseResult = await request.json().catch(() => null);
if (!bodyParseResult) {
  return NextResponse.json(
    { error: "Request body required" },
    { status: 400 }
  );
}

const parseResult = confirmationSchema.safeParse(bodyParseResult);
if (!parseResult.success) {
  return NextResponse.json(
    {
      error: "Confirmation required",
      message: "To delete your account, send { confirmDelete: true } in the request body",
      details: parseResult.error.format()
    },
    { status: 400 }
  );
}
```
4. Remove the old manual check code

Why z.literal(true): Ensures the value is EXACTLY true (not truthy), prevents confirmDelete: "yes" or confirmDelete: 1

Add Zod schemas to validate inputs for all server actions.

Steps:
1. Import Zod at top: `import { z } from "zod";`
2. Define schemas for each action:
```typescript
// After imports, before first function
const chatModelSchema = z.object({
  model: z.string().min(1, "Model name is required")
});

const titleMessageSchema = z.object({
  message: z.object({
    id: z.string(),
    role: z.enum(["user", "assistant", "system"]),
    content: z.string()
  })
});

const messageIdSchema = z.object({
  id: z.string().uuid("Invalid message ID format")
});

const chatVisibilitySchema = z.object({
  chatId: z.string().uuid("Invalid chat ID format"),
  visibility: z.enum(["public", "private"])
});
```
3. Add validation to each action:

**saveChatModelAsCookie:**
```typescript
export async function saveChatModelAsCookie(input: { model: string }) {
  const { model } = chatModelSchema.parse(input);
  const cookieStore = await cookies();
  cookieStore.set("chat-model", model);
}
```

**generateTitleFromUserMessage:**
```typescript
export async function generateTitleFromUserMessage(input: { message: UIMessage }) {
  const { message } = titleMessageSchema.parse(input);
  // rest of function unchanged
}
```

**deleteTrailingMessages:**
```typescript
export async function deleteTrailingMessages(input: { id: string }) {
  const { id } = messageIdSchema.parse(input);
  // rest of function unchanged

```

**updateChatVisibility:**
```typescript
export async function updateChatVisibility(input: {
  chatId: string;

  // rest of function unchanged

```

Why parse instead of safeParse: Server actions throw errors which Next.js catches and returns to client as error state. Using parse() throws ZodError with validation details.

Type safety: Keep existing TypeScript parameter types, Zod provides runtime validation
