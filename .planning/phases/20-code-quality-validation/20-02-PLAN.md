Add Zod schema for agent creation in POST /api/admin/agents.

Steps:
1. Import Zod at top of file: `import { z } from "zod";`
2. Define schema before POST handler:
```typescript
const createAgentSchema = z.object({
  fullName: z.string().min(1, "Full name is required").max(255),
  email: z.string().email("Invalid email format").toLowerCase(),
  phoneNumber: z.string().optional(),
  region: z.enum(["paphos", "limassol", "larnaca", "nicosia", "famagusta", "all"], {
    errorMap: () => ({ message: "Invalid region" })
  }),
  role: z.string().min(1, "Role is required"),
  isActive: z.boolean().optional().default(true),
  notes: z.string().optional(),
});
```
3. Replace manual validation (lines 173-178) with:
```typescript
const parseResult = createAgentSchema.safeParse(body);

  return NextResponse.json(
    {
      error: "Validation failed",
      details: parseResult.error.format()
    },
    { status: 400 }
  );
}
const validatedData = parseResult.data;
```
4. Use `validatedData` throughout the handler instead of `body`

Steps:
1. Import Zod at top of file: `import { z } from "zod";`
2. Define schema before PUT handler (all fields optional since this is partial update):
```typescript
const updateAgentSchema = z.object({
  fullName: z.string().min(1).max(255).optional(),
  email: z.string().email("Invalid email format").toLowerCase().optional(),
  phoneNumber: z.string().optional(),
  region: z.enum(["paphos", "limassol", "larnaca", "nicosia", "famagusta", "all"]).optional(),
  role: z.string().min(1).optional(),
  isActive: z.boolean().optional(),
  canReceiveLeads: z.boolean().optional(),
  canUpload: z.boolean().optional(),
  listingOwnerEmail: z.string().email().optional(),
  landline: z.string().optional(),
}).strict(); // Reject unknown fields
```
3. Add validation after auth checks (before line 127 where body is used):
```typescript
const parseResult = updateAgentSchema.safeParse(body);
if (!parseResult.success) {
  return NextResponse.json(
    {
      error: "Validation failed",
      details: parseResult.error.format()
    },
    { status: 400 }
  );
}
const validatedData = parseResult.data;
```
4. Update the field mapping section (lines 158-172) to use `validatedData` instead of `body`
5. Ensure email conflict check (lines 141-154) uses `validatedData.email`

Why .strict(): Prevents clients from sending unexpected fields that would be silently ignored
Why all optional: PUT is a partial update, any subset of fields can be provided
