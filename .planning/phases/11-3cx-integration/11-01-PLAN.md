  <action>
Create the `ThreeCXClient` class that handles authentication to the 3CX system.

**3CX has two common API patterns depending on version:**

1. **3CX v18+ REST API** (preferred): POST to `/api/login` with JSON body `{ Username, Password }`, returns a session cookie or bearer token
2. **3CX Web Client API**: POST to `/webclient/api/Login/GetAccessToken` with form data

**Implementation approach:**
- Create `ThreeCXClient` class that accepts `ThreeCXConfig`
- `login()` method:
  - Try the v18+ REST API first: POST `{baseUrl}/api/login` with `{ Username: config.username, Password: config.password }`
  - If that returns 404, fall back to web client API: POST `{baseUrl}/webclient/api/Login/GetAccessToken`
  - Store the authentication token/cookie from response
  - Set `this.authenticated = true` on success
  - Throw descriptive error on failure (invalid credentials vs network error vs unknown endpoint)
- `isAuthenticated()` method: returns boolean
- `getAuthHeaders()` method: returns headers dict with auth token for subsequent requests
- `makeAuthenticatedRequest(path: string, options?: RequestInit)`: wrapper that adds auth headers and handles 401 (re-login attempt)
- Private `_token: string | null` field
- Private `_cookie: string | null` field (some 3CX versions use cookies)

**Error handling:**
- Use structured logging via `logger` from sophia-bot utils
- On auth failure: log error with category, throw `ThreeCXAuthError` (custom error class, define at top of file)
- On network failure: log with retry info, throw with descriptive message
- On session expiry (401 during request): attempt one re-login, then throw if still failing

**Key pattern from existing code to follow:**
- Use `withRetry` from `../sophia-bot/utils/retry.ts` for the login fetch call
- Use `logger` from `../sophia-bot/utils/logger.ts` with `LogCategory.GENERAL` (or consider adding a new category if appropriate)
- Keep the client stateful but lightweight (no persistent connections needed in Edge Functions)

DO NOT use any browser-based libraries (Puppeteer, Playwright). This must work with plain `fetch()` in Deno.
