  <action>
Create the call log extraction service that fetches and filters 3CX call records.

**`extractTodayCalls(client: ThreeCXClient)` function:**
- Calculate today's date range in Cyprus timezone (Europe/Nicosia):
  - `startOfDay`: today at 00:00:00 Cyprus time, converted to UTC ISO string
  - `endOfDay`: today at 23:59:59 Cyprus time, converted to UTC ISO string
- Call the 3CX call log API endpoint via `client.makeAuthenticatedRequest()`:
  - **3CX v18+ API**: GET `{baseUrl}/api/calllog?dateFrom={start}&dateTo={end}&filter={targetNumber}`
  - **Alternative endpoint**: POST `{baseUrl}/api/activeCalls/getCallLog` with body `{ dateFrom, dateTo, filter }`
  - Try both endpoints, use whichever succeeds (3CX versions vary)
- Parse response into `ThreeCXCallLogEntry[]` from types
- Return the raw entries

**`filterExternalCallers(entries: ThreeCXCallLogEntry[])` function:**
- Import `AUDIT_CONFIG` from config
- Filter entries where:
  - `calledNumber` matches `AUDIT_CONFIG.TARGET_NUMBER` (or contains it -- 3CX may include country code prefix)
  - `callerNumber` is NOT in `AUDIT_CONFIG.INTERNAL_EXTENSIONS`
  - `direction` is "Inbound" (only incoming calls to the main line)
- For each remaining entry, normalize the caller phone number using `normalizePhoneNumber()`
- Deduplicate phone numbers (same caller calling multiple times = one entry)
- Return `CallAuditResult` with:
  - `date`: today's date string (YYYY-MM-DD)
  - `totalCalls`: total entries before filtering
  - `externalCallers`: deduplicated normalized phone numbers
  - `internalFiltered`: count of filtered-out internal calls
  - `errors`: any parsing errors encountered

**`normalizePhoneNumber(phone: string)` function:**
- Strip whitespace, dashes, parentheses
- Handle Cyprus phone formats:
  - Local: "22032770" -> "+35722032770"
  - With country code: "+35722032770" -> "+35722032770"
  - With leading zero: "0 22 032770" -> "+35722032770"
  - International: "+44..." -> "+44..." (keep as-is)
- Return normalized string (always with + prefix and country code)
- If number is clearly invalid (too short, non-numeric after cleanup), log warning and skip

**Logging:**
- Log total calls fetched, internal filtered count, and external caller count
- Log any entries that fail to parse (malformed data from 3CX)
- Use logger with LogCategory.GENERAL

**IMPORTANT:** The 3CX API response format varies by version. Make the parser resilient:
- Try to map known field names (Caller, CallerNumber, caller_number, etc.)
- Log the raw response structure on first call for debugging
- If response format doesn't match expected shape, log full response (redacting phone numbers) and throw descriptive error
