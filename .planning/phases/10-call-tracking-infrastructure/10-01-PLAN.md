**Table 1: `call_audit_runs`** — Tracks each daily audit execution to prevent duplicate runs.
```sql
CREATE TABLE call_audit_runs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  audit_date DATE NOT NULL,                    -- The date being audited (e.g., 2026-02-26)
  status TEXT NOT NULL DEFAULT 'running',       -- running, completed, failed
  total_calls INTEGER DEFAULT 0,               -- Total external calls found
  missing_callers INTEGER DEFAULT 0,           -- Callers not found in Telegram groups
  error_message TEXT,                          -- Error details if failed
  started_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  completed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(audit_date)                           -- Prevents duplicate processing of same day
);
CREATE INDEX idx_audit_runs_date ON call_audit_runs(audit_date DESC);
CREATE INDEX idx_audit_runs_status ON call_audit_runs(status);
```

**Table 2: `call_records`** — Stores individual call records extracted from 3CX for each audit run.
```sql
CREATE TABLE call_records (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  audit_run_id UUID NOT NULL REFERENCES call_audit_runs(id) ON DELETE CASCADE,
  caller_phone TEXT NOT NULL,                  -- Normalized phone number (E.164 or local format)
  call_time TIMESTAMPTZ NOT NULL,             -- When the call happened
  target_number TEXT NOT NULL DEFAULT '22032770', -- The main line number called
  is_internal BOOLEAN NOT NULL DEFAULT FALSE,  -- Whether this was an internal extension
  found_in_telegram BOOLEAN DEFAULT NULL,      -- NULL=not checked, true=found, false=not found
  telegram_group TEXT,                         -- Which group it was found in (if any)
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_call_records_audit_run ON call_records(audit_run_id);
CREATE INDEX idx_call_records_phone ON call_records(caller_phone);
CREATE INDEX idx_call_records_found ON call_records(found_in_telegram) WHERE found_in_telegram IS NULL OR found_in_telegram = FALSE;
```

**Table 3: `caller_alerts`** — Tracks alert state and follow-up for each missing caller phone number.
```sql
CREATE TABLE caller_alerts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  caller_phone TEXT NOT NULL,                  -- The phone number that wasn't found
  audit_run_id UUID NOT NULL REFERENCES call_audit_runs(id) ON DELETE CASCADE,
  call_record_id UUID REFERENCES call_records(id) ON DELETE SET NULL,
  status TEXT NOT NULL DEFAULT 'pending',       -- pending, alerted, follow_up_sent, resolved, ignored
  alert_message_id TEXT,                       -- Telegram message ID of the alert sent
  follow_up_message_id TEXT,                   -- Telegram message ID of 24h follow-up
  resolution_type TEXT,                        -- found_in_group, vasya_confirmed, alternative_number, manual_resolve
  resolution_note TEXT,                        -- Additional context (e.g., "Found by Vasya in Limassol group")
  alternative_phone TEXT,                      -- If Vasya provides an alternative number
  alerted_at TIMESTAMPTZ,                      -- When initial alert was sent
  follow_up_at TIMESTAMPTZ,                    -- When follow-up reminder was sent
  resolved_at TIMESTAMPTZ,                     -- When the alert was resolved
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE(caller_phone, audit_run_id)           -- One alert per phone per audit run
);
CREATE INDEX idx_caller_alerts_status ON caller_alerts(status);
CREATE INDEX idx_caller_alerts_phone ON caller_alerts(caller_phone);
CREATE INDEX idx_caller_alerts_follow_up ON caller_alerts(status, alerted_at)
  WHERE status = 'alerted';                    -- For finding alerts needing 24h follow-up
```

**Types to define at the top of the file:**
```typescript
// Audit run status
type AuditRunStatus = "running" | "completed" | "failed";

// Alert status progression: pending -> alerted -> follow_up_sent -> resolved/ignored
type AlertStatus = "pending" | "alerted" | "follow_up_sent" | "resolved" | "ignored";

// Resolution types
type ResolutionType = "found_in_group" | "vasya_confirmed" | "alternative_number" | "manual_resolve";

// Audit run record
interface AuditRun {
  id: string;
  audit_date: string;
  status: AuditRunStatus;
  total_calls: number;
  missing_callers: number;
  error_message: string | null;
  started_at: string;
  completed_at: string | null;
}

// Call record from 3CX extraction
interface CallRecord {
  id: string;
  audit_run_id: string;
  caller_phone: string;
  call_time: string;
  target_number: string;
  is_internal: boolean;
  found_in_telegram: boolean | null;
  telegram_group: string | null;
}

// Caller alert for missing numbers
interface CallerAlert {
  id: string;
  caller_phone: string;
  audit_run_id: string;
  call_record_id: string | null;
  status: AlertStatus;
  alert_message_id: string | null;
  follow_up_message_id: string | null;
  resolution_type: ResolutionType | null;
  resolution_note: string | null;
  alternative_phone: string | null;
  alerted_at: string | null;
  follow_up_at: string | null;
  resolved_at: string | null;
  created_at: string;
  updated_at: string;
}
```

**Functions to implement:**

1. **`claimAuditRun(auditDate: string): Promise<AuditRun | null>`**
   - Attempts atomic INSERT into call_audit_runs with the given date
   - Uses unique constraint on audit_date to prevent duplicates (same pattern as `claimMessageForProcessing` in db.ts)
   - Returns the created AuditRun if successful, null if already exists (23505 error code)
   - On other errors, throw (fail-closed — don't process if state is uncertain)
   - Log with `LogCategory.DATABASE`, operation: "claimAuditRun"

2. **`getAuditRunByDate(auditDate: string): Promise<AuditRun | null>`**
   - SELECT from call_audit_runs WHERE audit_date = auditDate
   - Returns the run or null

3. **`completeAuditRun(runId: string, totalCalls: number, missingCallers: number): Promise<void>`**
   - UPDATE call_audit_runs SET status='completed', total_calls, missing_callers, completed_at=NOW()
   - WHERE id = runId

4. **`failAuditRun(runId: string, errorMessage: string): Promise<void>`**
   - UPDATE call_audit_runs SET status='failed', error_message, completed_at=NOW()
   - WHERE id = runId

5. **`saveCallRecords(auditRunId: string, records: Array<{caller_phone: string, call_time: string, target_number?: string, is_internal?: boolean}>): Promise<CallRecord[]>`**
   - Bulk INSERT into call_records
   - Map each record to include audit_run_id
   - Return inserted records (use .select() after insert)
   - Log count of records saved

6. **`updateCallRecordTelegramStatus(recordId: string, found: boolean, groupName?: string): Promise<void>`**
   - UPDATE call_records SET found_in_telegram=found, telegram_group=groupName
   - WHERE id = recordId

7. **`createCallerAlert(params: {caller_phone: string, audit_run_id: string, call_record_id?: string}): Promise<CallerAlert | null>`**
   - INSERT into caller_alerts
   - Handle 23505 (duplicate) gracefully — return null if alert already exists for this phone+run
   - Return created alert on success

8. **`updateAlertStatus(alertId: string, status: AlertStatus, extra?: {alert_message_id?: string, follow_up_message_id?: string, resolution_type?: ResolutionType, resolution_note?: string, alternative_phone?: string}): Promise<void>`**
   - UPDATE caller_alerts with the new status and any extra fields
   - Set alerted_at=NOW() when status changes to 'alerted'
   - Set follow_up_at=NOW() when status changes to 'follow_up_sent'
   - Set resolved_at=NOW() when status changes to 'resolved' or 'ignored'
   - Always set updated_at=NOW()

9. **`getUnresolvedAlerts(): Promise<CallerAlert[]>`**
   - SELECT from caller_alerts WHERE status IN ('pending', 'alerted', 'follow_up_sent')
   - ORDER BY created_at ASC

10. **`getPendingFollowUps(hoursThreshold: number = 24): Promise<CallerAlert[]>`**
    - SELECT from caller_alerts WHERE status = 'alerted' AND alerted_at < NOW() - interval '{hoursThreshold} hours'
    - These are alerts that were sent but haven't been resolved after the threshold
    - ORDER BY alerted_at ASC

11. **`getAlertByPhone(callerPhone: string, auditRunId: string): Promise<CallerAlert | null>`**
    - SELECT from caller_alerts WHERE caller_phone = callerPhone AND audit_run_id = auditRunId
    - Returns single alert or null
