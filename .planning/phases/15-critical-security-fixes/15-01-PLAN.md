**SEC-01 Fix: Password Hash Truncation**

Bcrypt generates 60-character hashes but the password column is varchar(64). This allows storage but risks future hash format changes. Expand to varchar(255) to accommodate all bcrypt variants.

1. Update schema definition:
   - File: `lib/db/schema.ts` line 24
   - Change: `password: varchar("password", { length: 64 })` → `password: varchar("password", { length: 255 })`

2. Create migration SQL:

   - Content:
```sql
-- Fix password hash column truncation (WA-001)
-- Bcrypt hashes are 60 chars but future hash formats may be longer

```

3. Apply migration:

**SEC-02 Fix: Chat Creation Race Condition**

Current implementation uses simple INSERT which fails on concurrent requests for same WhatsApp conversation. Add `onConflictDoNothing()` to make idempotent.

1. Update `saveChat()` function (line 88-111):
   - Add Drizzle's conflict handling after `.values()`:
```typescript
export async function saveChat({
  id,
  userId,
  title,
  visibility,
}: {
  id: string;
  userId: string;
  title: string;
  visibility: VisibilityType;
}) {
  try {
    return await db.insert(chat).values({
      id,
      createdAt: new Date(),
      userId,
      title,
      visibility,
    }).onConflictDoNothing();  // <-- ADD THIS LINE
  } catch (error) {
    log.error("Failed to save chat", error, { chatId: id, userId });
    throw new ChatSDKError("bad_request:database", "Failed to save chat");
  }
}
```

2. Verify `onConflictDoNothing` is imported:
   - Check top of file has: `import { onConflictDoNothing } from "drizzle-orm/pg-core"`
   - If not, add to existing drizzle-orm import block

Note: This relies on the existing PRIMARY KEY constraint on chat.id. Duplicate inserts with same id will silently succeed (no error thrown), preventing race condition crashes.
