Create `audit-pipeline.ts` with a single exported `runDailyAudit()` function that orchestrates the full pipeline:

**Function signature:**
```typescript
export async function runDailyAudit(dateOverride?: string): Promise<AuditPipelineResult>
```

**Return type:**
```typescript
interface AuditPipelineResult {
  success: boolean;
  auditRunId: string | null;
  date: string;
  totalCalls: number;
  externalCallers: number;
  missingCallers: number;
  alertsSent: number;
  alertsFailed: number;
  skippedReason?: string; // e.g., "duplicate_run", "no_external_callers"
  errors: string[];
}
```

**Pipeline steps (in order):**

1. **Calculate audit date** â€” Use `AUDIT_CONFIG.TIMEZONE` ("Europe/Nicosia") to determine today's date in YYYY-MM-DD format. If `dateOverride` provided, use that instead.

2. **Claim audit run** â€” Call `claimAuditRun(auditDate)` from call-tracking.ts. If returns null (duplicate), return early with `skippedReason: "duplicate_run"` and `success: true`.

3. **Wrap remaining steps in try/catch** â€” On failure, call `failAuditRun(runId, errorMessage)` before rethrowing.

4. **Authenticate and extract calls** â€” Create `ThreeCXClient`, call `client.login()`, then `extractTodayCalls(client)`, then `filterExternalCallers(entries)`. Save call records: `saveCallRecords(runId, records)` where records is mapped from the raw entries.

5. **Search Telegram groups for each caller** â€” For each phone in `externalCallers`:
   - Call `searchPhoneInGroups(phone, Object.values(REGIONAL_GROUP_IDS))`.
   - If found: update call_record's telegram status via `updateCallRecordTelegramStatus(recordId, true, groupName)`.
   - If NOT found: add to `missingCallers` array.
   - Wrap in try/catch per caller â€” one failure shouldn't abort the entire audit.
   - **IMPORTANT:** If REGIONAL_GROUP_IDS contains any 0 values (unconfigured), skip the Telegram search step entirely and log a warning. Do NOT throw â€” the 3CX extraction is still valuable without Telegram search.

6. **Send alerts for missing callers** â€” For each missing caller:
   - Call `createCallerAlert({ caller_phone, audit_run_id: runId })` from call-tracking.ts.
   - Call `sendMissingCallerAlert({ phoneNumber, callTime, callDate, searchedGroups })` from telegram-alerts.ts.
   - If alert send succeeds: `updateAlertStatus(alertId, { status: "alerted", alert_message_id: String(messageId) })`.
   - Use 1-second delay between sends (rate limiting already handled by sendMissingCallerAlert, but add explicit delay between createCallerAlert + sendMissingCallerAlert pairs).
   - **IMPORTANT:** If ZYPRESS_OTHERS_CHAT_ID is 0 (unconfigured), skip alert sending entirely and log a warning. Do NOT throw.

7. **Complete audit run** â€” Call `completeAuditRun(runId, totalCalls, missingCallers.length)`.

8. **Return result** â€” Populate all fields of AuditPipelineResult.

**Error handling pattern:** Wrap the entire pipeline (steps 4-7) in try/catch. On error, call `failAuditRun(runId, error.message)` and return `{ success: false, errors: [error.message], ... }`. Do NOT rethrow â€” the Edge Function should always return a JSON response.

**Imports needed:**

**Logging:** Log at each major step with `logger.info(...)`. Use structured logging fields consistent with existing patterns (category: LogCategory.GENERAL, operation: "runDailyAudit").
