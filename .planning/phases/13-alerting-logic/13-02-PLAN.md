<objective>
Create the follow-up reminder system that automatically re-alerts for unresolved missing callers after 24 hours, and integrate it into the daily audit pipeline.

Purpose: Without follow-ups, alerts can be forgotten. This ensures unresolved callers get a reminder so no lead falls through the cracks. The follow-up runs at the end of each daily audit, catching any alerts from the previous day that haven't been resolved.

Output: `follow-up.ts` module + updated pipeline integration
</objective>

  <action>
Create `follow-up.ts` with the follow-up reminder logic:

**Function signature:**
```typescript
export async function processFollowUpReminders(): Promise<FollowUpResult>
```

**Return type:**
```typescript
interface FollowUpResult {
  checked: number;        // Total alerts checked
  remindersSent: number;  // Successfully sent reminders
  remindersFailed: number;
  skippedReason?: string; // e.g., "telegram_not_configured"
}
```

**Logic:**

1. **Guard: Check Telegram is configured** — If `ZYPRESS_OTHERS_CHAT_ID === 0`, return early with `skippedReason: "telegram_not_configured"`.

2. **Query pending follow-ups** — Call `getPendingFollowUps(24)` from call-tracking.ts. This returns `CallerAlert[]` where status is `alerted` and `alerted_at` is older than 24 hours.

3. **For each pending alert:**
   a. Calculate `daysSinceAlert` = Math.floor((Date.now() - new Date(alert.alerted_at).getTime()) / (24 * 60 * 60 * 1000)).
   b. Create `MissingCallerInfo` from the alert fields: `{ phoneNumber: alert.caller_phone, callTime: "N/A", callDate: alert.created_at.split("T")[0], searchedGroups: [] }`.
      - Note: The original call_time isn't stored in caller_alerts, so use "N/A" or reconstruct from the associated call_record if available. For simplicity, use the alert's created_at date.
   c. Format the reminder: `formatFollowUpReminder(callerInfo, daysSinceAlert)`.
   d. Send via the Telegram bot directly (import `getTelegramBot` from `../_shared/telegram.ts` and `ZYPRESS_OTHERS_CHAT_ID` from `../_shared/telegram-search.ts`):
      ```typescript
      const bot = getTelegramBot();
      const result = await bot.sendMessage({ chatId: ZYPRESS_OTHERS_CHAT_ID, text: reminderText });
      ```
   e. On success: `updateAlertStatus(alert.id, { status: "follow_up_sent", follow_up_message_id: String(result.message_id) })`.
   f. On failure: log error but continue to next alert.
   g. Add 1-second delay between messages (rate limiting).

4. **Return result** — Populate FollowUpResult.

**Imports:**
- `getPendingFollowUps, updateAlertStatus` from `../_shared/call-tracking.ts`
- `formatFollowUpReminder, MissingCallerInfo` from `../_shared/telegram-alerts.ts`
- `getTelegramBot` from `../_shared/telegram.ts`
- `ZYPRESS_OTHERS_CHAT_ID` from `../_shared/telegram-search.ts`
- `logger, LogCategory` from `../sophia-bot/utils/logger.ts`

**Error handling:** The entire function is wrapped in try/catch. If getPendingFollowUps throws, return `{ checked: 0, remindersSent: 0, remindersFailed: 0 }` with the error logged. Individual send failures do NOT abort the loop.

**Update audit-pipeline.ts:**

1. Import `processFollowUpReminders` from `./follow-up.ts`.
2. Add a new field to `AuditPipelineResult`:
   ```typescript
   followUp?: {
     checked: number;
     remindersSent: number;
     remindersFailed: number;
     skippedReason?: string;
   };
   ```
3. After step 7 (completeAuditRun) and before the final return, add:
   ```typescript
   // Step 8: Process follow-up reminders for stale alerts from previous days
   try {
     const followUpResult = await processFollowUpReminders();
     result.followUp = followUpResult;
     logger.info("[Audit Pipeline] Follow-up reminders processed", {
       category: LogCategory.GENERAL,
       operation: "runDailyAudit",
       ...followUpResult,
     });
   } catch (followUpError) {
     // Follow-up failures should NOT fail the entire audit
     logger.error("[Audit Pipeline] Follow-up processing failed", followUpError instanceof Error ? followUpError : new Error(String(followUpError)), {
       category: LogCategory.GENERAL,
       operation: "runDailyAudit",
     });
   }
   ```
   The follow-up step is AFTER completeAuditRun so the day's audit is already recorded even if follow-ups fail.

**Update index.ts:**

Add a new query parameter `?follow-up-only=true` that ONLY runs follow-up reminders without doing the full 3CX audit. This is useful for testing or if follow-ups need to be triggered independently.

const isFollowUpOnly = url.searchParams.get('follow-up-only') === 'true';
