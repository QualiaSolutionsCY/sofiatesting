## Milestone Summary

**Key Decisions:**
- Zod validation before switch statement for early rejection of malicious payloads
- TextEncoder for byte-accurate payload size measurement (multi-byte UTF-8)
- Separate .eq() queries instead of .or() string interpolation to prevent filter injection
- Module-level loadingPromise variable for single-inflight-request pattern
- Map-based sliding window rate limiting (sufficient for serverless)
- PostgreSQL json_agg + COALESCE for JOIN-based message aggregation

**Issues Resolved:**
- Password hash truncation risk (varchar(64) → varchar(255))
- Chat creation race condition (INSERT ON CONFLICT DO NOTHING)
- Hardcoded dev9.zyprus.com URL (environment variable)
- Email enumeration via registration endpoint (timing-safe responses)
- Tool argument injection (Zod schemas with runtime constraints)
- SQL filter injection via .or() interpolation (parameterized queries)
- Admin prompt endpoint accepts unlimited payloads (50KB limit)
- Prompt cache thundering herd (single inflight request)
- File upload abuse (per-user rate limiting)
- Data export N+1 queries (JOIN optimization)

**Issues Deferred:**
- Per-tool rate limiting (EF-008) → v1.4
- OAuth token validation in Zyprus client (EF-009) → v1.4
- AbortController timeout on image downloads (EF-010) → v1.4
- CSRF protection (WA-010) → v1.4
- Generic error messages in production (WA-011) → v1.4
- Pagination limits (WA-012) → v1.4
- Code deduplication (WA-013) → v1.4
- Hardcoded UUIDs (WA-014) → v1.4
