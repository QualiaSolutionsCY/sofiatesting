# Milestone v1.6: Upload Pipeline Production Hardening

## Context

AI review on 2026-03-20 found 17 issues in Sophia's property upload pipeline (WhatsApp + Email). 4 CRITICAL, 6 HIGH, 4 MEDIUM, 3 LOW. Agents are hitting real bugs: upload lock leaks, broken email follow-ups, silently dropped fields, and business rules not enforced.

## Goal

Fix all CRITICAL and HIGH findings so the upload pipeline is reliable for both WhatsApp and email channels. Zero silent data loss, zero lock leaks, zero dropped business rules.

## Functional Requirements

### FR-1: Upload Lock Lifecycle (CRITICAL #1)
Release upload lock on ALL early-return paths in field-validation.ts, not just the duplicate detection path. Currently 6+ paths leak the lock, blocking agents for the full TTL.

### FR-2: listingType Validation (CRITICAL #3)
Add `listingType` to `validateRequiredFields` required array. Remove `.optional()` from Zod schema. A listing without sale/rent must fail, not silently proceed.

### FR-3: Email Google Maps Follow-up (CRITICAL #4)
When email upload blocks on missing Google Maps link, the response must be stored in chat_history AND the follow-up email must load that history so the agent can reply with the link.

### FR-4: parsePreExtractedFields Robustness (CRITICAL #2)
Log a warning when the regex fails to match. Make the regex more resilient to formatting variations.

### FR-5: Email Bedrooms Preservation (HIGH #6)
Remove `bedrooms` from the `nullableFields` list in ai-chat.ts so the AI's correct `bedrooms: 0` for studios is not deleted.

### FR-6: poolType Parsing Fix (HIGH #7)
Handle the `poolType: "none" [warning text]` format in parsePreExtractedFields. Strip text after the closing quote.

### FR-7: Michelle Rental Reassignment (HIGH #8)
Actually apply `specialCase.modifiedRequest.assignTo` in field-validation.ts when `handleSpecialCases` returns it.

### FR-8: Bedrooms Default Removal (HIGH #9)
Remove `.default(0)` from bedrooms Zod schema. Let missing bedrooms fail validation properly.

### FR-9: Double clearPendingImages Cleanup (HIGH #5)
Remove the redundant clearPendingImages call in ai-chat.ts (property-listing.ts already handles it).

### FR-10: condition Field Passthrough (MEDIUM #14)
Pass `args.condition` to `createDraftListing` so agent-provided property condition isn't silently dropped.

### FR-11: extractAssignmentFromEmail Patterns (MEDIUM #12)
Add "assign this to" and "assign listing to" patterns.

### FR-12: ToolResult Interface Dedup (LOW #16)
Single ToolResult definition shared between field-validation.ts and executor.ts.

## Non-Functional Requirements

### NFR-1: No Regression
All existing upload tests (WhatsApp + email) must still pass after changes.

### NFR-2: Deploy Verified
Deploy sophia-bot edge function and verify with at least one WhatsApp test upload and one email test upload.

## Definition of Done

- [ ] All 12 FRs implemented
- [ ] `npx tsc --noEmit` passes (N/A for Deno — manual type check)
- [ ] sophia-bot deployed to production
- [ ] WhatsApp test upload succeeds
- [ ] Email test upload succeeds
- [ ] No upload lock leaks observable in logs
