# Milestone v1.1: Reliability & Hardening

**Status:** SHIPPED 2026-01-29
**Phases:** 6-9
**Total Plans:** 16

## Overview

Eliminate system fragility — consolidate prompts, validate images, improve cache management, harden error handling. This milestone completed the LIST-06 WhatsApp image upload requirement carried from v1.0.

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Logging first | Enables debugging all other phases | Good |
| MAX(updated_at) for version tracking | No migration needed | Good |
| x-admin-secret header for admin auth | Simple, effective | Good |
| is_current pattern for version tracking | Clearer than soft delete | Good |
| Templates priority 80 | After cyprus_knowledge at 70 | Good |
| Rollback creates new version | Append-only, never mutates history | Good |
| Validate at ingress | Fail fast, users get immediate feedback | Good |
| HEAD request with GET fallback | Server compatibility | Good |

**Issues Resolved:**

- 5-minute cache delay during development (admin invalidation endpoint)
- Same instruction in multiple prompts causes conflicts (conflict detection script)
- AI sometimes hallucinates image URLs (validation at ingress)
- Technical errors exposed to users (user-friendly message mapper)
- No way to revert bad prompt changes (rollback API)

**Issues Deferred:**

- Agent phone numbers required for WhatsApp testing (documented in STATE.md)
- SOPHIA_ADMIN_SECRET must be set after deployment (documented)

**Technical Debt Incurred:**

- correlation_id column migration for pending_images (backward-compatible code handles missing column)
