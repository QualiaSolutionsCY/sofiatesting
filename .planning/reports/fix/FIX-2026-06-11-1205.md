# Fix Report - 2026-06-11 12:05

**Symptom:** Bank-portal listings publish, but the office managers of that area never get the publish confirmation on WhatsApp. (Lauren, Zyprus, 2026-06-11)
**Mode:** general
**Outcome:** fixed

## Feedback Loop
- Initial inspection: traced the publish-confirmation recipient from `listing-notifier` → `listing_uploads.agent_phone` → the `notifyPhone` resolution in both upload handlers; confirmed against live `agents` data.
- Final passing check: `pnpm exec vitest run` — no NEW failures vs baseline (10 pre-existing `reviewer-assignment.test.ts` failures unrelated to this change, proven by stash + re-run); live `agents` query confirms each region resolves to exactly one reachable manager.

## Root Cause
`agents` rows for `request{region}@zyprus.com` carry **placeholder mobiles** `+35799000001`–`+35799000005` (sequential, nobody owns them).

- `supabase/functions/sophia-bot/tools/handlers/property-listing.ts:580` — for a bank listing, `reviewers.listingOwner` is the regional-office email, so the confirmation was redirected to `getAgentByEmail(office).mobile` = a placeholder number → the WhatsApp went into the void.
- `supabase/functions/sophia-bot/tools/handlers/land-listing.ts:910` — the owner-redirect was gated on `args.assignTo`, which bank listings never set, so bank **land** listings notified the *uploader* instead of the office — never the managers.

Either way, the **region's managers** (the people who actually own bank listings) were never notified.

## Patch
- Files changed: 4
  - `supabase/functions/sophia-bot/agents/identifier.ts` — new `getOfficeManagers(regionalOfficeEmail)`: inverts `REGIONAL_EMAILS` to recover the region, returns `role='manager'` agents for it (office accounts are `role='agent'`, so placeholders are excluded by construction).
  - `supabase/functions/sophia-bot/tools/handlers/property-listing.ts` — bank branch routes the confirmation to the region's managers (comma-joined phones for multi-manager regions).
  - `supabase/functions/sophia-bot/tools/handlers/land-listing.ts` — same bank branch, and the `args.assignTo`-only guard is now an `else if` so bank land listings redirect too.
  - `supabase/functions/listing-notifier/index.ts` — `agent_phone` may be a comma-separated list; fan out `sendTextMessage` to each reachable recipient. Single-value (non-bank) rows split to a 1-element list — backward compatible.
- Data change (production `agents` table, project `vceeheaxcrhmpqueudqx`): `diana@zyprus.com` demoted `manager → agent` per Lauren ("Diana is not a manager anymore"). The fix reads roles live, so this removes her from Limassol bank-listing confirmations automatically. Limassol now resolves to Michelle Longridge only.

Why 4 files (default max 3): one root cause with three reachable code seams (two upload paths + the notifier) plus the shared helper that both paths call — kept deliberately free of unrelated churn.
