# Fix Report - 2026-06-28 13:26

**Symptom:** Sophia replies "I'm experiencing technical difficulties right now. Please try again in a few moments." to ordinary messages (e.g. "Create invoice") when the OpenRouter account is near its spend cap ($667.61 of a $700 limit). "Hi" succeeded at 10:04:30 UTC; "Create invoice" failed at 10:04:37 UTC — 7s apart, near-identical request shape.
**Mode:** quick
**Outcome:** fixed (transient-limit resilience). The hard outage itself is account-level — resolved out-of-band by topping up OpenRouter credit.

- Failure path confirmed at `services/ai-chat.ts:702-708` (both primary + fallback OpenRouter calls returned error/empty → the line-705 user message).
- Final check: `supabase functions deploy sophia-bot` bundles + type-checks under Deno (deno not installed locally; deploy is the authoritative type-check per project memory `tsc-skips-deno-edge`).

## Root Cause
`services/ai-chat.ts:35-37` + `:442-451` — PRIMARY_MODEL = PRO_MODEL = FALLBACK_MODEL = `anthropic/claude-sonnet-4.6`, so the fallback at `:688` hits the **identical** model/provider and cannot escape a 429/account-level limit. The 429 retry used a **blind exponential backoff with no jitter and ignored the server's `Retry-After`**, so concurrent WhatsApp webhooks retried in lockstep and re-triggered the same rate limit, then fell through to the "technical difficulties" message. (The circuit breaker is NOT the cause — `utils/circuit-breaker.ts` self-heals OPEN→HALF_OPEN→CLOSED within the 60s `resetTimeoutMs`.)

## Patch
- Files changed: `supabase/functions/sophia-bot/services/ai-chat.ts` (1 file)
- Change summary: In the 429 branch of `callOpenRouter`, honor the server's `Retry-After` header when present (capped at 8s so a long wait can't blow the Edge Function's ~120s wall), and add up to 500ms of jitter so concurrent webhooks de-synchronize their retries. Falls back to the original exponential backoff when no `Retry-After` is sent. No model/quality change.

A genuine escape from an OpenRouter account-level 429/402 needs a *second provider* — e.g. Anthropic-direct (`ANTHROPIC_API_KEY`) for the same Sonnet 4.6, or a second OpenRouter key. This keeps quality identical (respects CLAUDE.md "no free-tier models for Sophia") while removing the single point of failure.
