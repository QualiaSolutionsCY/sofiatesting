# Fix Report - 2026-07-01 15:10

**Symptom:** Edited a monthly (Moayad Fawzi) / yearly source invoice and saved, but the change did not reflect on the UPCOMING instance in the recurring-run preview.
**Mode:** quick

## Feedback Loop
- Initial failing check: inspection — the detail-pane inline edit DOES update `lineItems`/`amount` and reconciles into `docs` (`App.tsx:1040,1050`), so `recurringRows` recomputes with the edit; yet the MonthlyRun preview stayed stale.
- Final passing check: `npx tsc --noEmit` = 0; invoice unit tests 49/49; logic trace of the draft-restore path.

## Root Cause
`components/invoices/redesign/overlays/MonthlyRun.tsx:162-184` — on open, the overlay seeds `edited` from the live rolled-forward source (`recurringRows`), then unconditionally layers the localStorage delivery-draft on top, re-applying `description`/`sub`. So a stale overlay draft (or a source-invoice edit made AFTER a draft was saved) overrode the ledger edit — the upcoming preview showed the old value.

## Patch
- Files changed: `components/invoices/redesign/overlays/MonthlyRun.tsx` (1 file)
- Change summary: record the source value each source-derived draft field (`description`/`sub`) was diffed against (`descriptionBase`/`subBase`), stamped in both `flushDrafts` and the debounced `editRow` save. On reopen, restore a draft's description/amount ONLY when the current source still matches that base; if the operator has since edited the monthly/yearly invoice, the source wins and the stale draft field is dropped. Drafts saved before the fix have no base → treated as stale, so the source shows through. `recipients`/`message` (overlay-only) still always restore, so in-overlay auto-save is preserved.
