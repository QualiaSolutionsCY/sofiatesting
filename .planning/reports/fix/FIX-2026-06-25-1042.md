# Fix Report - 2026-06-25 10:42

**Symptom:** (1) On a normal invoice **approve**, Sophia asked "what message should I send to the accounting group?" — Marios: no message for normal invoices, only credit notes. (2) On an **explicit commission** request ("commission for flat rental…"), Sophia created the draft WITHOUT asking which agent — wrong.
**Mode:** quick
**Outcome:** fixed (deployed)

## Why it kept breaking (the real root cause)
The `autoresearch` prompt-optimizer (pg_cron every 6h) was rewriting the DB `invoicing` prompt overnight → the flow "worked last night" then broke. The prior fix added `"invoicing"` to `FILE_OVERRIDE_KEYS` (prompt-loader.ts), so the optimizer can no longer touch it. From now on the invoicing prompt only changes via the file + redeploy — it stays fixed.

## Patch (invoicing prompt only — 1 file)
`supabase/functions/sophia-bot/prompts/behaviors/invoicing.ts`:
- **approve:** never ask "what message?" for an invoice. Normal invoice → approve with NO groupMessage (blank caption). Commission invoice → approve with groupMessage = the agent's name. Group-message question is for credit notes ONLY.
- **commission (refined):** commission is detected ONLY by the agent literally writing "commission". If commission → ask ONE question ("Which agent made the sale/rental?") BEFORE create, remember the name, pass it as groupMessage at approval. If NOT commission (plain rental/services) → create immediately, never ask about an agent. (Replaces the prior over-broad "never ask which agent" that wrongly skipped the agent on real commissions.)
