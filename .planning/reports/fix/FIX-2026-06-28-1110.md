# Fix Report - 2026-06-28 11:10

**Symptom:** "Send invoice no 11450 to my email" → Sophia: "I wasn't able to locate invoice № 11450 in the system to email it." Emailing an invoice by its official number fails (no PDF delivered).
**Mode:** quick (1 file)
**Outcome:** fixed

## Root Cause
`lib/invoices/sophia/intent-handlers.ts` `findDoc()` only matched against `p.documentId`. The manageInvoice tool has a SEPARATE `officialNumber` param, and for "invoice no 11450" Sophia passes the bare number there (or as "№ 11450") — which `findDoc` never read, so every number-referenced intent (email_invoice, query_status, resend…) returned "couldn't find". This was masked before: the earlier flow emailed via the generic `sendEmail` (the saved "last document"), which never called `findDoc`. Forcing `email_invoice` (the duplicate-email fix) exposed the gap.

## Patch
- `lib/invoices/sophia/intent-handlers.ts` (1): `findDoc` now matches by `documentId` OR `officialNumber`, normalized (strips a leading "№ ", trims) — so "11450", "№ 11450", and "INV-2026-11450-DRAFT" all resolve.
