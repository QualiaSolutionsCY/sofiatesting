## Root Cause
`components/invoices/redesign/ledger/DetailPane.tsx:550` (and the "accounting" delivery card, ~612-638) — the admin **Delivery** panel's "Zyprus Accounting" card was tied to `doc.stage === "sent-to-accounting"` (the *paid* state) and rendered the message **"Payment received · {№} · receipt {receiptNo} attached. From Marios C., via Sophia."** with a "Sent" badge on both WhatsApp + email channels.

This is purely a **UI representation bug**: it told Marios the receipt was delivered to the accounting group. No actual send occurs.

### Why no real send exists (verified)
- Real group sends go through `sendDocumentToAccountingGroup()` — called ONLY for invoices (`approve`, `edit_invoice`, `notifyAccountingGroupOfInvoiceAction`, `resendCorrectedInvoiceAction`) and credit notes (`notifyGroupOfCreditNote`). **Never for receipts.** (`lib/invoices/actions/documents.ts`)
- `markPaidAndIssueReceiptAction` comment + code: *"Receipts go to MARIOS ONLY — never posted to the accounting group."* (`actions/documents.ts:404`)
- Edge-fn `handleManageInvoice` sends the receipt PDF to the **requester's own number** (`phoneNumber || agent.mobile`), not a group (`supabase/functions/sophia-bot/tools/handlers/invoice.ts:112`).
- File-fallback invoicing prompt: *"receipts are never posted to the group"* (`invoicing.ts:44`). `sophia_prompts` table currently has no `invoicing` row, so the (correct) file fallback is live.
- `forwardAccountingAction` → `queueAccountingHandoff` → `deliverWithManualProvider` returns `deliveryStatus: "manual-copy-ready", copied: false` — it does NOT auto-send a WhatsApp.

## Patch
- Files changed: `components/invoices/redesign/ledger/DetailPane.tsx` (1 file)
- Change summary:
  - `accountingDelivered` now = `!isReceipt && hasNumber` — the accounting group receives the **invoice/credit note on approval**, never a receipt.
  - The accounting card for a **receipt** now reads "Receipts are delivered to Marios only — they are never sent to the accounting group", shows channels as "Not sent", and drops the "Resend both" action (Configure only).
  - For an invoice/credit note, the card now correctly says it was "sent to the accounting group on approval" (was wrongly framed as on-payment with the receipt attached).
