## Root Causes
- **#9** `services/image-handler.ts:194` — `checkImageAccessibleWithError` sent `HEAD`/`GET` with **no User-Agent**, so bank-portal CDNs (REMU) reject valid images with 404. The GET fallback only triggered on 405, never 404. The project's own `portal-scraper.ts:378 fetchPageHtml` already sends a browser UA for this exact reason — the validator did not.
- **#10** `services/description-generator.ts:1625 generateLandDescription` pushed 4 interior blank lines (`lines.push("")`) where normal listings (`generateDescription`) use only one blank before the CTA.
- **#7** `prompts/behaviors/property-upload.ts:575` already said "default to separate for land" but did not forbid *asking* — Sophia still printed "Title Deed Status: Unknown — can you confirm?" and asked.
- **#8** No prompt rule existed for bank-portal auto-assignment; `formatPortalSummary` (portal-scraper.ts:986) even referenced a non-existent "BANK-PORTAL UPLOAD RULES above". The handler (`special-cases.ts:53` / `land-listing.ts:299`) asks management for assignment whenever `assignTo` is absent, so without Sophia passing `assignTo`, the question always fires.
