# Fix Report - 2026-06-29 13:27

**Symptom:** On a commission invoice, when the agent gives a first name only (e.g. "Andreas") for the commission agent, Sophia improvises "What is Andreas's surname? I need the full name for the commission record." Marios: no surname needed — accept whatever name is given (one or two words).
**Mode:** quick (prompt). **Outcome:** fixed

## Root Cause
`supabase/functions/sophia-bot/prompts/behaviors/document-routing.ts:68-72` — the "CRITICAL: Full Name Requirement" tells Sophia ALL person names need first + surname and to ask "What is [Name]'s surname?". It lists buyer/vendor/client/seller, but Sophia over-generalised it to the commission AGENT name (which the invoicing prompt only collects as the accounting-group label, never a document field). document_routing (priority 30) outranks invoicing (45), so the surname demand won.

## Patch (1 file)
- `document-routing.ts` — added an explicit EXCEPTION right under the Full Name Requirement: the commission agent name is NOT a document person field (only the group label, never printed), the full-name rule does NOT apply, accept whatever is given (one word or two, first-name-only fine), and NEVER ask for the agent's surname/full name.
