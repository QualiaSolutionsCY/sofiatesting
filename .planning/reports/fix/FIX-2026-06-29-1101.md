# Fix Report - 2026-06-29 11:01

**Symptom:** On "correction & resend", the typed REASON FOR CORRECTION (e.g. "test1") was NOT sent to the accounting group — the group always got the default "Corrected invoice N — please use this version. Ignore the previous one." Expected: typed reason → group gets that reason; empty → default.
**Mode:** quick (backend) — invoice correction-resend flow.
**Outcome:** fixed

## Root Cause
`lib/invoices/actions/documents.ts` — both correction paths hardcoded the group caption to the default notice and DISCARDED the operator's reason by design (`resendCorrectedInvoiceAction` :726-727, `correctResendAction` :396-397; comments literally said "The correction reason is NEVER placed on a caption"). The reason reached the action (stored as correctionReason, App.tsx:924 passes it) but was never used as the caption.

## Patch (1 file)
- `lib/invoices/actions/documents.ts` — both functions now build the group caption as: typed reason (trimmed) when present, else the default correction notice. `resendCorrectedInvoiceAction` uses the existing `trimmed`; `correctResendAction` uses `reason?.trim()`.

## Verification
- `npx tsc --noEmit` — documents.ts PASS
- Live: operator to edit an already-sent invoice with a reason and confirm the group caption matches (pending deploy).

## Remaining Notes
- Reason is sent verbatim as the caption (replaces the default), matching the operator's expectation. Empty reason still sends the default notice. The corrected PDF still attaches in both cases.
