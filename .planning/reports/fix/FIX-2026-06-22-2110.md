# Fix Report - 2026-06-22 21:10

**Symptom:** Invoices / receipts / credit-notes created over WhatsApp (via Sophia) don't appear in the admin panel — "I created some invoices and nothing reflected."
**Mode:** general
**Outcome:** fixed

## Feedback Loop
- Initial failing check: queried `invoice_documents` directly — the WhatsApp-created docs (e.g. "Hossam Hasan" invoice 11436 + receipt 10103 + credit-note 10394, created 17:20–17:25) ARE in the DB; a fresh panel load shows all 29. So the write path works; the open panel just never re-fetches.
- Final passing check: `npx tsc --noEmit` (exit 0) + refetch reuses the same `loadDocumentsAction` loader that a reload uses (proven to show all 29 docs).

## Root Cause
`app/invoices/page.tsx` loads documents **once** server-side (`force-dynamic`) and `components/invoices/redesign/App.tsx` had **no client-side re-fetch**. Documents created out-of-band (over WhatsApp → `/api/sophia/intent` → same `invoice_documents` table) therefore only appeared after a full manual page reload. With the SPA panel left open, nothing reflected.

## Patch
- Files changed:
  - `components/invoices/redesign/App.tsx` — import `loadDocumentsAction`; add `refetchDocuments()` (re-pull + `reconcile`, selection preserved); add an on-focus / `visibilitychange` effect that refetches when the operator returns to the tab (skipped while the composer is open so an in-progress draft is never disrupted); pass `onRefresh` to the topbar.
  - `components/invoices/redesign/chrome/Topbar.tsx` — add a manual **Refresh** button (`RefreshCw`) wired to `onRefresh`.
- Change summary: the panel now pulls out-of-band documents on tab focus and via an explicit Refresh button, without a full reload. Non-disruptive: DetailPane re-seeds its inline form only on `doc.id` change, so a same-selection refetch preserves in-progress edits; the composer is guarded.

## Verification
- `npx tsc --noEmit` - PASS (exit 0)
- Topbar importers check: only `App.tsx` consumes it - PASS (no other caller broken by the new prop)

## Remaining Notes
- No interval polling was added (deliberately) to avoid wiping an unsaved local "duplicate" draft or disrupting edits — focus + manual button cover the reported case safely. A guarded poll can be added later if live auto-update is wanted.
