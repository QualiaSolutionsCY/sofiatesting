## Root Cause
- **Symptom 1** — `tools/handlers/field-validation.ts:504` and `tools/handlers/land-listing.ts:306` ask "who to assign" via `needsAssignmentInput()` / `handleSpecialCases()`, both of which run **before** `isBankListing` is computed (was field-validation:588 / land-listing:366) and **neither was bank-aware**. So for a management agent (Lauren), the handler returned the assignment question before the bank auto-assign (`assignReviewers(..., isBankListing)`) was ever reached. `needsAssignmentInput` (`rules/reviewer-assignment.ts:128`) ignored `isBankListing` entirely.
- **Symptom 2** — `tools/handlers/data-retrieval.ts:229` hard-blocked `extractFromBankPortal` for any non-management agent ("only available to management… ask Lauren or Charalambos").

## Patch (5 files, one cohesive change)
- `rules/reviewer-assignment.ts` — `needsAssignmentInput(agent, type, isBankListing?)`: returns `false` for bank listings.
- `rules/special-cases.ts` — `UploadRequest.isBankListing`; the management "who to assign?" branch now also requires `!request.isBankListing`.
- `tools/handlers/field-validation.ts` — compute `isBankListing` BEFORE the assignment questions; pass it to `handleSpecialCases` + `needsAssignmentInput`; drop the now-duplicate later declaration.
- `tools/handlers/land-listing.ts` — same reordering + pass-through.
- `tools/handlers/data-retrieval.ts` — remove the management-only gate on bank-portal extraction (any agent may upload; ownership auto-routes to the regional office).

Net behaviour: a bank link (property OR land) from ANY agent → no "who to assign?" question; listing owner = `request{region}@zyprus.com` for the property's region (existing `assignReviewers` bank branch, now actually reached). The AI must pass `bankUrl` (already mandated by the MUST-COPY block + DB prompt v11); the handler is now the deterministic gate that no longer forces the question.
