## Root Cause (per reported issue)
- **Duplicate images** — `services/portal-scraper.ts:949` `mergeStructured()` assigned `scraped.imageUrls.filter(...)` directly. Every OTHER image path de-dupes through a `Set` (raw-HTML :478, markdown :742, deterministic gallery :1023); this Firecrawl-extract merge path did not, so repeated gallery URLs survived on listings where the deterministic gallery (<3 matches, e.g. REMU) does not override.
- **Agent avatar leaks** — exclusion regex `services/portal-scraper.ts:953` only drops URLs containing `logo|avatar|icon|...`. A bank agent photo served from the property CDN without those tokens is NOT caught. Needs a real avatar URL to extend the pattern — speculative without it.
- **Missing internal photos** — gallery is collected by per-portal CDN regex (:1024-1042); "missing internal" = some gallery images don't match the portal pattern. Needs the live HTML to extend regex — speculative without it.
- **NO POOL when description has a pool** — DELIBERATE. Extraction prompt (`:248`) instructs the model to NOT extract a pool unless it appears in a Features/Amenities section, explicitly ignoring the free-text description, to kill the Firecrawl "common pool" hallucination. `groundFeatures()` (:1057) would keep a description pool, but the model never extracts it. This is a product tradeoff (anti-hallucination vs. completeness), lives in DB prompt v11, and needs a decision before flipping.
- **Veranda / energy / land-size** — already extracted by the current prompt (`:194-217`) as of the 2026-06-09 commit; no code gap found.
