## Root Cause
The live `invoicing` prompt is `sophia_prompts` row id `d7ebb93e`, **version 4, `is_current=true`, `updated_by: "autoresearch:gen2"`**. The **prompt-optimizer (autoresearch, pg_cron every 6h)** auto-rewrote the invoicing prompt and injected a "Commission invoices — the agent name is mandatory / Which agent made the sale?" section + VAT-confirmation framing. The optimizer keeps regenerating it → "same same same" recurrence. (The 3 older rows v1–v3 are clean; the file fallback is clean.)

## Patch (invoicing only)
- `supabase/functions/sophia-bot/services/prompt-loader.ts` — added `"invoicing"` to `FILE_OVERRIDE_KEYS` so the loader ALWAYS uses the file version and **autoresearch can no longer change invoicing behavior** (this is the documented anti-auto-optimize lever, same as `safety_rules`).
- `supabase/functions/sophia-bot/prompts/behaviors/invoicing.ts` — added a forceful "NEVER improvise extra questions" rule: have client+amount → create_draft immediately; "with VAT"/"+VAT"/"plus VAT" = vatMode plus (also the default), never re-ask VAT; NEVER ask about commission/agent unless the agent explicitly says "commission"; a rental is not a commission.
