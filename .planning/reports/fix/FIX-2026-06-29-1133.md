# Fix Report - 2026-06-29 11:33

**Symptom:** Editing an already-sent invoice and clicking "Correct & resend" (or Save) sometimes throws "Something went wrong" (React error boundary). After it, the content shows saved but "Correct & resend" is gone, so the correction never reaches the group.
**Mode:** quick (frontend). **Outcome:** fixed (crash) + diagnosed (disappearing button).

## Root Cause
`components/invoices/redesign/App.tsx` — `onUpdateDoc` (Save) and `onCorrectResendDoc` (Correct & resend) ran their server actions inside `startTransition` with NO try/catch. `onCorrectResendDoc` does updateDocumentAction (saves) THEN resendCorrectedInvoiceAction (WhatsApp group/Marios send). The group send intermittently throws (the Wasender token 401), and the unhandled throw crashes the whole app. Because the save already persisted, the invoice afterward shows "saved" (no unsaved changes) and the "Correct & resend" button is gone — the correction never reached the group. Reproduced live: a successful run did 2× POST 200 (no crash); the crash only occurs when the send throws.

## Patch (1 file)
- `App.tsx` — wrapped both handlers' transitions in try/catch. On failure: a clear toast ("Saved, but couldn't notify the group: …" / "Couldn't save: …") + refetchDocuments() instead of an app crash. Mirrors the approve/cancel auto-refresh fix.

## Verification
- `npx tsc --noEmit` — App.tsx PASS
- Live reproduction confirmed the crash path (unhandled throw) and a success path (200/200)

## Remaining Notes
- "Correct & resend disappears after a quiet Save" is by design (Save = save without notifying; Correct & resend = save + notify). With this fix a failed send no longer crashes and tells the operator to retry. A persistent "resend correction to group" action for an already-saved invoice would be a separate small feature if wanted.
- The underlying intermittent Wasender 401 (finding #3) is left per the operator's earlier instruction; this fix makes the UI resilient to it.
