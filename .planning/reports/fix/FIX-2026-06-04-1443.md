# Fix Report - 2026-06-04 14:43

**Symptom:** "the pdf should be produced the same template as the admin panel" — invoice PDFs Sophia sends over WhatsApp (and the in-app "Download PDF") looked nothing like the polished invoice the admin panel renders.
**Mode:** frontend
**Outcome:** fixed

## Feedback Loop
- Initial inspection: two PDF paths diverged. `window.print()` of the HTML `TemplatePreview` component = the nice admin template; `buildDocumentPdfBytes` (`lib/invoices/pdf.ts`) = a flat text-only PDF used by BOTH Sophia's `attachPdf` and the admin "Download PDF".
- Final passing check: rendered sample PDF (`buildDocumentPdfBytes`) to PNG via `pdftoppm` — visually matches the admin template; `pdfinfo` parses cleanly (1 page); `/Length` byte-accurate; `npx tsc --noEmit` exits 0.

## Root Cause
`lib/invoices/pdf.ts:8` (old `buildDocumentPdfBytes`) — emitted a bare list of text lines (Helvetica, no letterhead, no table, no totals/IBAN), while the admin panel's real template is the styled HTML `components/invoices/redesign/ledger/TemplatePreview.tsx` printed by the browser. The two never shared a renderer, so the stored/WhatsApp PDF and the on-screen template were entirely different documents.

## Patch
- Files changed: `lib/invoices/pdf.ts` (single file).
- Change summary: rewrote `buildDocumentPdfBytes` to render the template's structure as a real styled PDF — letterhead (entity name, address, Reg/VAT No.), document title + meta (number/date/due/billing), BILL TO block, bordered line-item table with gray header (#, Description, Qty, Unit · €, Total · €), totals (Subtotal / VAT % / Total due), SETTLEMENT block (Beneficiary/IBAN/Bank) with receipt + credit-note variants, and a NOTE block. Added Helvetica/Helvetica-Bold advance-width tables for right-alignment + word-wrap, WinAnsi encoding (so the € glyph renders and xref offsets stay byte-accurate), and an adaptive header divider that clears a 3–4 row meta block. No new dependency. Fixes both consumers (Sophia `attachPdf` → `storage.ts` → `buildDocumentPdfBytes`, and admin `downloads.ts`) at once.
