# Fix Report - 2026-06-23 12:30

**Symptom:** Ensure Marios receives his copy when an invoice is approved over WhatsApp (Sophia).
**Mode:** general
**Outcome:** fixed (already delivered by the send-retry; hardened further by Marios-first ordering)

## Root Cause
- The WhatsApp approve handler (`lib/invoices/sophia/intent-handlers.ts:211`) already calls
  `notifyMariosApprovedAction` unconditionally, so Marios is sent his numbered copy on every approve.
  The earlier silent-drop root cause was the WhatsApp document SEND having no retry
  (`lib/whatsapp/client.ts`), fixed in commit a398119 — that fix covers this path too (same client).

## Patch
- Files changed: `lib/invoices/sophia/intent-handlers.ts`
- Send Marios's copy BEFORE the accounting-group copy on approve, so a rate-limited trailing send
  can only affect the group copy, never Marios's. Both sends remain retry-hardened.

- `npx tsc --noEmit` - PASS
- Path trace: approve → notifyMariosApprovedAction → notifyMariosOverWhatsApp → marios.msisdn (retry).
