# Fix Report - 2026-06-28 11:31

**Symptom:** Credit-note documents show the source invoice number TWICE — once as a meta line "Applies to: Invoice N" and again in the line-item description "Credit note for invoice no N". Marios wants the redundant "Applies to" line removed.
**Mode:** quick (PDF + preview, 2 files — kept in sync)
**Outcome:** fixed

## Root Cause
`lib/invoices/pdf.ts:135` pushed an "Applies to: Invoice N" entry into the credit-note meta block, and `components/invoices/redesign/ledger/TemplatePreview.tsx:77-82` rendered the matching "Applies to" row — duplicating the number that already appears in the description line.

## Patch
- `lib/invoices/pdf.ts` — dropped the `meta.push(["Applies to:", …])` for credit notes.
- `components/invoices/redesign/ledger/TemplatePreview.tsx` — removed the matching "Applies to" preview row (PDF↔preview parity).

## Verification
- `npx tsc --noEmit` — PASS (0)
- `npx next build` — PASS (0)
- Visual: download a credit note → the "Applies to: Invoice N" line is gone; the number remains in the description.
