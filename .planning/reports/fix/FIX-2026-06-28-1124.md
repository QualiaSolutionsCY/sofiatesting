# Fix Report - 2026-06-28 11:24

**Symptom:** After approval, the accounting GROUP gets the approved PDF correctly, but Marios's DIRECT copy never arrives ("Sent Marios his copy" is reported, nothing lands).
**Mode:** quick (2 files — one per runtime)
**Outcome:** fixed (route Marios's copy through the bot's working 1:1 sender)

## Root Cause
Two senders exist. The EDGE bot delivers 1:1 PDFs via `sendDocumentByUrl` (WASEND) — the channel Marios actually has an open chat with (he DMs the bot to approve), so it reliably lands. Vercel's `notifyMariosApprovedAction` uses a DIFFERENT `wasenderClient` (WASENDER) that posts to the group but cannot DM Marios — it returns success without delivering, so the reply falsely says "Sent Marios his copy". Additionally `storeDocumentPdfAction` threw when its post-upload metadata save failed, dropping the approved PDF's signed URL (so `attachPdf` returned empty on approve → no `pdfUrl` for the bot to forward).

## Patch
- `lib/invoices/actions/documents.ts` — `storeDocumentPdfAction` now returns the signed URL even if the secondary metadata save throws (upload already succeeded), so `attachPdf` reliably yields `pdfUrl` on approve.
- `supabase/functions/sophia-bot/tools/handlers/invoice.ts` — after a successful `approve`, the bot ALSO sends the approved PDF to Marios (35799921560) via `sendDocumentByUrl` (the working 1:1 sender), blank caption. Skips when Marios is the approver (no double-send).
