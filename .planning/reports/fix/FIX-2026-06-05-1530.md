# Fix Report - 2026-06-05 15:30

**Symptom:** A newly created invoice (e.g. one Marios creates) doesn't appear at the top of the admin-panel invoice list.
**Mode:** general
**Outcome:** fixed

## Feedback Loop
- Initial failing check: code inspection — both the adapter and the list pane re-sorted by `issued` (issue date) descending, so a new invoice dated today sorted *below* invoices with future issue dates (the seeded recurring invoices carry issue dates like 06/27, 06/29).
- Final passing check: `npx tsc --noEmit` → exit 0. List now inherits repository order (`updated_at desc`), so the most recently created/updated invoice is first.

## Root Cause
`lib/invoices/redesign/adapter.ts:97` and `components/invoices/redesign/ledger/ListPane.tsx:39` both sorted by `b.issued.localeCompare(a.issued)` (issue date desc). `listInvoiceDocuments()` already returns rows ordered `updated_at desc` (`lib/invoices/supabase/document-repository.ts:40`), and `saveInvoiceDocument(s)` both `return listInvoiceDocuments()` — so the freshest invoice is always first in the data, but the two UI sorts discarded that order in favour of issue date, burying any new invoice whose issue date wasn't the latest.

## Patch
- Files changed: `lib/invoices/redesign/adapter.ts`, `components/invoices/redesign/ledger/ListPane.tsx` (2 files)
- Change summary: removed both `issued`-based sorts; the list now preserves the repository's `updated_at desc` order so newly created/updated invoices land on top. Added comments explaining the contract.

## Verification
- `npx tsc --noEmit` - PASS (exit 0)

The "Approved" group (and other non-Draft groups) show 0 because all 30 invoices are currently `status: "draft"` — expected given the data, not a filter bug.
