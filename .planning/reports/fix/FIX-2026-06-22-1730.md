# Fix Report - 2026-06-22 17:30

**Symptom:** Issuing a receipt over WhatsApp is slow and sends Marios the receipt PDF twice.
**Mode:** perf / general
**Outcome:** fixed

## Feedback Loop
- Initial failing check: code inspection of the Sophia receipt path (duplicate send + double PDF build)
- Final passing check: `npx tsc --noEmit` (exit 0)

## Root Cause
`lib/invoices/actions/documents.ts:336` — `markPaidAndIssueReceiptAction` unconditionally calls
`notifyMariosOverWhatsApp(issuedReceipt)` (builds the PDF + sends it over WhatsApp). In the Sophia/WhatsApp
path the bot ALSO delivers the receipt PDF to the requester via `sendDocumentByUrl`
(`supabase/functions/sophia-bot/tools/handlers/invoice.ts`), after `attachPdf` builds + stores the PDF a
second time (`lib/invoices/sophia/intent-handlers.ts:312`). Net: **2 PDF builds + 2 WhatsApp sends** on the
critical path, and when Marios is the requester he receives the receipt **twice** — and Sophia's reply only
returns after all of it completes, so the reply is slow.

## Patch
- Files changed:
  - `lib/invoices/actions/documents.ts` — `markPaidAndIssueReceiptAction(id, { notifyMarios })`; default
    `true` (admin panel keeps its send), guarded `if (notifyMarios) await notifyMariosOverWhatsApp(...)`.
  - `lib/invoices/sophia/intent-handlers.ts` — receipt intent now calls the action with
    `{ notifyMarios: false }`, so the bot's own delivery is the single send.
- Change summary: the WhatsApp receipt path now does one PDF build + one send. The admin-panel path is
  unchanged (still notifies Marios). No recipient regression: the bot delivers the PDF to the requester.
