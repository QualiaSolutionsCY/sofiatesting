# Fix Report - 2026-06-11 12:40

**Symptom:** On a bank **land** listing, Sophia doesn't add the Google Maps location link (a house/property listing for the same bank did). (Lauren, Zyprus, 2026-06-11)
**Mode:** general
**Outcome:** fixed

## Feedback Loop
- Initial inspection: compared the Google-Maps-link handling between `property-listing.ts` and `land-listing.ts`.
- Final passing check: `pnpm exec vitest run` — 326 passed, only the 10 pre-existing `reviewer-assignment.test.ts` failures (baseline, unrelated).

## Root Cause
`supabase/functions/sophia-bot/tools/handlers/land-listing.ts:827` — the Google Maps link was added to My Notes **only when a user-supplied `locationUrl` exists**, with no fallback to the resolved coordinates, and the land confirmation message (`:976`) had **no maps line at all**. Property uploads (`property-listing.ts:690-694`) fall back to `https://www.google.com/maps/place/{lat},{lon}` from `resolvedCoordinates`. Bank land listings carry portal coordinates but no user `locationUrl`, so they fell into the gap → no Google location anywhere.

## Patch
- Files changed: 1 — `supabase/functions/sophia-bot/tools/handlers/land-listing.ts`
  - My Notes: added `else if (resolvedCoordinates)` fallback that builds the Maps link from coordinates (mirrors property).
  - Confirmation message: added a `📍 Google Maps:` line (user link, else coordinates), matching the property confirmation the agent sees.

- Deployed `sophia-bot` to prod.
