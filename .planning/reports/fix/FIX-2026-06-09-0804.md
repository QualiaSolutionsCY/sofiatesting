## Root Causes (deterministic — fixed)
1. **Title deed wrong** — `tools/handlers/data-retrieval.ts:259` **hardcoded** `title_deed_status: "pending"` for every bank-portal upload; `zyprus/property-types.ts` maps "pending" → "Not Available". So properties with a real separate deed showed "Not Available". → Now set from the page's own deed wording (newly extracted), else `"unknown"` + ask the agent. Never hardcoded.
2. **Location defaults to "Agios Nektarios"** — `zyprus/locations.ts:474` silently returned `taxonomy.locations[0]` (the first location in the Zyprus API response) for any unmatched, district-less location. → Removed; falls through to the documented `DEFAULT_LOCATION_UUID` placeholder, and the AI is told to confirm the area.
3. **Location empty for Altia/REMU** — `services/portal-scraper.ts` had regex backfills for price/beds/area/coords but **none for location**, and Altia/REMU URLs carry no location. → Added a district-anchored location backfill (5 Cyprus districts, can't grab junk).
4. **Year of build never captured** — not in the scraper `EXTRACTION_SCHEMA` (though the upload tool has `yearBuilt`, `schemas.ts:154`). → Added `yearBuilt` to schema + Firecrawl merge + markdown backfill + MUST-COPY (`year_built`).
5. **"common pool" hallucination** — Firecrawl's features prompt invited example amenities and the MUST-COPY block forced verbatim copy. → Tightened the Firecrawl `features` prompt ("only explicitly-listed features; never add a pool unless the page lists one") + added an anti-invention rule to the bank-portal upload rules + a guard line in `formatPortalSummary`.
