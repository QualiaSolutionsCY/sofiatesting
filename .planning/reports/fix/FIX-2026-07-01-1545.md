# Fix Report - 2026-07-01 15:45

**Symptom:** (1) A just-created invoice (Mohammad Ahmad) doesn't appear in the admin panel without a manual Refresh. (2) The created draft can't be seen under "All Invoices" (only under Draft). (3) Marios wants to edit the client-delivery message/email BEFORE approving (he may tweak the default before the approve fires the email + accounting-group send).
**Mode:** quick (3 small, related fixes)
**Outcome:** fixed

## Feedback Loop
- Initial failing check: inspection — no `setInterval` polling in App.tsx (only on-focus/visibility refresh); ListPane sort puts numbered invoices before drafts; DetailPane client-delivery `msg` returns "Disabled until the invoice is numbered." for drafts.
- Final passing check: `npx tsc --noEmit` = 0; invoice unit tests 49/49.

## Root Cause
1. `components/invoices/redesign/App.tsx:197-210` — invoices are only re-pulled on tab focus / visibility change and the manual Refresh button; nothing pulls out-of-band (Sophia/WhatsApp) creations while the panel sits open + focused.
2. `components/invoices/redesign/ledger/ListPane.tsx` sort — numbered invoices sort first, then drafts, so a fresh draft lands at the very bottom of "All Invoices" (buried under 70+ numbered rows).
3. `components/invoices/redesign/ledger/DetailPane.tsx:670-678` — the client-delivery card message showed "Disabled until the invoice is numbered." for drafts instead of the letterhead default, so Marios couldn't preview/edit the message before approving (the Edit buttons were already enabled).

## Patch
- Files changed: `App.tsx`, `ledger/ListPane.tsx`, `ledger/DetailPane.tsx` (3 files)
- Change summary:
  1. App.tsx — add a 15s `setInterval` poll → `refetchDocuments()`, guarded on operator present, composer closed, and tab visible. Ledger stays live without a manual Refresh.
  2. ListPane.tsx — sort DRAFTS first (newest, pending) then numbered by number desc, so a just-created invoice is visible at the top of "All Invoices".
  3. DetailPane.tsx — always show the letterhead default (`clientDeliveryMessage`) in the client-delivery card, even on a draft, so Marios can preview + edit it (Edit message / Edit email already work on drafts) before approving; only "Send email" still waits for the official number.
