## Root Cause
`lib/invoices/actions/documents.ts:357` — `correctResendAction` marked the doc corrected (`markCorrectedForResend`), saved it, and called `queueCorrectedResend` (which only writes an internal **manual-delivery record** via `queueManualDelivery`). It **never called `sendDocumentToAccountingGroup`**, so the corrected invoice never reached the accounting WhatsApp group — yet the admin UI toasts "Resent correction." The sibling `resendCorrectedInvoiceAction` (line 663) does it correctly. The admin stage-action button (`App.tsx:343`) and the legacy `InvoiceDashboard.tsx:425` both use the broken `correctResendAction`.

## Patch
- Files changed: `lib/invoices/actions/documents.ts` (1)
- Change summary: after the save + delivery-record queue, `correctResendAction` now sends the corrected PDF to the accounting group via `sendDocumentToAccountingGroup` and notifies Marios via `notifyMariosOverWhatsApp`, with an "ignore the previous one" caption — mirroring `resendCorrectedInvoiceAction`. `markCorrectedForResend` already flags the PDF `needs-regeneration`, and the group sender rebuilds the PDF from the corrected doc, so the group gets the new content. Sends are best-effort (never break the save). Fixes every caller of `correctResendAction` in one change.
