## Root Cause
`lib/invoices/supabase/document-repository.ts:42` ‚Äî `throw new Error("Unable to load invoice documents: ...")`.

The `/invoices` route (`app/invoices/page.tsx:10`, a `force-dynamic` Server Component) awaits `loadDocumentsAction()` ‚Üí `listInvoiceDocuments()`. Production has `INVOICE_SUPABASE_URL` + `INVOICE_SUPABASE_SERVICE_ROLE` set (Vercel, 15d ago), so it runs in **supabase mode** and queries the `invoice_documents` table on project **`tijadsdysuxkxrpdlecq`**.

That Supabase project **no longer exists** ‚Äî its subdomain has been removed from DNS (`getaddrinfo`/curl exit 6, HTTP 000). A *paused* project still resolves DNS; a missing DNS record means the project was **deleted**. The query therefore fails, line 42 throws, and the throw is uncaught in the server component ‚Üí 500 RSC render error.

Production log evidence: `2026-06-17 11:55:23 error Œª GET /invoices 500 Error: Unable to load invoice doc‚Ä¶` (matches the literal throw string).
